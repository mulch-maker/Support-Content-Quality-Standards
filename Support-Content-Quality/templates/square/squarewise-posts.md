This table outlines each section in the procedure template and how/when to use it.

## Style Details

### Headings and Sections

#### Title

**For Procedures**
- Posts that give advocates step-by-step instructions for completing a task and resolving a problem
- Style: Verb-centric
- Example: Verify Sellers by Phone

**For Overviews**
- Posts that cover broad internal topics like "Seller Verification Guidelines"
- Style: Noun-centric
- Example: Seller Verification Guidelines

**Geo tagged**
- Lists the country (or geo) applicable seller geo
- Style: In brackets, listed alphabetically
- Example: Verify Sellers by Messaging [AU, CA, ES, FR, IE, UK, US]
- If applicable to all geos, use [Global]

#### General information
- This is a mandatory heading that should remain on all posts

**Seller geographies**
- This section contains the "geo" or "market" that the post applies to
- This is determined by the country where the seller's account was activated
- Keep for all types of posts
- Delete any market that doesn't apply

**About this post**
- This is an optional section, used when an additional description is needed
- Can be used or deleted for any post

**Definitions**
- An optional section for any post type, containing definitions for frequently used words and phrases or general rules and requirements
- Delete this section if you aren't using it for your post

**Tools**
- Here you will include a list of the internal tools advocates may use to complete the procedural steps
- Always use in a post that includes a procedure
- Add or delete any tools necessary to the procedure
- Delete this section when it is not necessary in your post

#### Procedural steps and instruction lists (the body)

Use the body of the post as needed based on the post type.

**If procedural:**
- Use verb-centric headers or instructional lists
- Sentence case
- Example: Step 1: Verify the seller

#### Non-sequential topics (the body)

**If an overview:**
- Use noun-centric headers for non-sequential topics
- Sentence case
- Example: Verification guidelines

#### Revisions
After all changes have been loaded for publication, [document substantial and non-substantial procedural changes](https://squarewise.square.com/groups/57783/posts/5428480-manage-post-changes) in the Revisions section at the bottom of the post.

## Assets

### Callouts
- Provides extra context to a post
- Optional to use in any type of post
- Can be copied and pasted from the Asset section of the procedure template

### Tables
- Use in any post to provide visual structure and simplify complicated textual descriptions
- Tables can be copied and pasted from the Asset section of the procedure template
