# Content Quality Checklist

A comprehensive checklist to ensure Block support content meets quality standards before publication.

## 📋 Pre-Publishing Validation

Use this checklist to validate content against Block's support writing standards. Check all applicable items before publishing.

### ✅ Writing Principles Compliance

**One article, one purpose**
- [ ] Content addresses a single task, issue, or decision
- [ ] Title clearly indicates the specific purpose
- [ ] No unrelated topics or tangential information included

**Clear, concise, and purposeful**
- [ ] Uses plain language at 6th-8th grade reading level
- [ ] Active voice used wherever possible
- [ ] Gets to the point quickly without unnecessary complexity
- [ ] Word count appropriate for content type (see template guidelines)

**Structured to guide and empower**
- [ ] Starts with key context customers need
- [ ] Information organized logically
- [ ] Uses headings, bullet points, and lists effectively
- [ ] Content is scannable and easy to navigate

**Customer experience first**
- [ ] Leads with customer intent and needs
- [ ] Respects customers' time with efficient structure
- [ ] Enables customer independence and self-service
- [ ] Focuses on what customers want to accomplish

**Global audience design**
- [ ] Written for easy localization
- [ ] Avoids idioms, colloquialisms, and cultural references
- [ ] Uses consistent terminology throughout
- [ ] Considers different cultural contexts appropriately

**Cross-channel consistency**
- [ ] Maintains Block voice and tone standards
- [ ] Uses standardized templates and formats
- [ ] Aligns with messaging across other platforms

### ✅ Style Guidelines Adherence

**Links**
- [ ] Links full titles or surface names (not partial phrases)
- [ ] Avoids multiple hyperlinked phrases in single sentences
- [ ] No generic terms like "click here" or "read more"
- [ ] Uses descriptive text instead of full URLs
- [ ] All links tested and functional

**Instruction Lists and Steps**
- [ ] Uses numbered lists for sequential information only
- [ ] One action per step
- [ ] Begins steps with strong action verbs
- [ ] **Bolds** UI text that users interact with
- [ ] Uses exact names of buttons, menus, fields, system messages
- [ ] Present tense used throughout
- [ ] Sentence punctuation, no exclamation marks
- [ ] Maximum two levels of hierarchy
- [ ] Plain, direct language without jargon

**Apologies**
- [ ] No apology language in published content (unless genuine mistake)
- [ ] Focuses on solutions and encouragement rather than sympathy
- [ ] Avoids "we apologize for the inconvenience"
- [ ] Uses empowering language instead of sympathy

**Support Team Terminology**
- [ ] Uses "support" (lowercase) when referring to help/assistance
- [ ] Uses "[Business Name] Support" when referring to team interactions
- [ ] Consistent terminology for external-facing content
- [ ] Appropriate internal terminology for advocate-facing content

### ✅ Template Compliance

**Structure**
- [ ] Follows appropriate template structure completely
- [ ] All required sections included
- [ ] Optional sections used appropriately
- [ ] Maintains template formatting and style

**Content Components**
- [ ] Title follows template guidelines (length, format, style)
- [ ] Audience identification clear and accurate
- [ ] Context/prerequisites provided where needed
- [ ] Instructions clear and complete
- [ ] Outcome/success criteria specified
- [ ] Related resources included appropriately

### ✅ Business Unit Specific Requirements

**Square Content**
- [ ] Uses official Square product names correctly
- [ ] References Square Dashboard accurately with full navigation paths
- [ ] Specifies required permissions and access levels
- [ ] Considers different business types and subscription tiers
- [ ] Links to appropriate Square resources

**Cash App Content**
- [ ] Appropriate tone for financial service context
- [ ] Uses correct Cash App terminology and feature names
- [ ] Action links formatted correctly (messaging content)
- [ ] Dynamic fields tested and functional
- [ ] Compliance with financial services regulations
- [ ] Accessibility standards met (WCAG compliance)

### ✅ Technical Validation

**Functionality**
- [ ] All links tested and working
- [ ] Action links functional (Cash App messaging)
- [ ] Dynamic fields populate correctly
- [ ] Screenshots current and accurate
- [ ] Navigation paths verified in actual systems

**Accessibility**
- [ ] Proper heading hierarchy (H1, H2, H3)
- [ ] Alt text provided for images
- [ ] Color not used as sole means of conveying information
- [ ] Text contrast meets WCAG standards
- [ ] Content readable by screen readers

**SEO & Findability** (Web content)
- [ ] Title optimized for search
- [ ] Meta description compelling and accurate
- [ ] Keywords naturally integrated
- [ ] Internal linking strategy implemented
- [ ] URL structure follows conventions

### ✅ Review & Approval

**Content Review**
- [ ] Peer review completed
- [ ] Subject matter expert (SME) validation
- [ ] Legal/compliance review (if required)
- [ ] Accessibility review completed

**Final Checks**
- [ ] Spelling and grammar verified
- [ ] Formatting consistent throughout
- [ ] All placeholders replaced with actual content
- [ ] Version control updated
- [ ] Publishing workflow followed

## 🔄 Continuous Improvement

After publishing, track:
- [ ] Customer feedback and satisfaction
- [ ] Usage analytics and performance
- [ ] Support ticket reduction (if applicable)
- [ ] Advocate efficiency improvements (internal content)

Use this data to refine future content and update this checklist as needed.
