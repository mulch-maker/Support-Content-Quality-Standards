# Style Guides - Current Standards

Current writing principles and style guidelines documented across Block support content as of June 2026.

## 📋 Overview

These guidelines represent the documented standards currently in use across Block support communications, as captured from Writer's Block documentation and team practices.

## 📚 Style Guide Components

### [Writing Principles](./writing-principles.md)
**The 6 fundamental principles that guide all Block support content:**

1. **One article, one purpose** - Each piece addresses a single task or issue
2. **Be clear, concise, and purposeful** - Use plain language and active voice
3. **Structure content to guide and empower** - Organize logically with clear headings
4. **Write for the customer experience first** - Lead with customer intent and needs
5. **Design for a global audience** - Write for localization and accessibility
6. **Be consistent across channels** - Maintain unified voice and standards

### [Style Guidelines](./style-guidelines.md)
**Detailed style rules covering:**

- **Links** - How to hyperlink effectively and accessibly
- **Instruction Lists & Steps** - Writing clear, sequential instructions
- **Apologies** - When and how to use apology language appropriately
- **Support Team Terminology** - Consistent language for referring to teams and roles
